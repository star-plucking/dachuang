面向本科生创新创业训练计划项目的一站式材料生成 Skill。

不知道大创题目怎么起，申报书怎么写？  
项目方向有了，但开题、中期、结题和答辩材料总是断档？  
实验数据、成果包装、经费预算和进度记录不知道怎么组织？  
学校模板、指导老师意见和真实项目进展之间总是对不上？

把零散想法快速整理成大创全链路材料包的 Skill。  
可以围绕项目方向生成大创风格选题、开题报告、中期报告、结题报告、过程记录、答辩提纲、可选项目配图和成果包装占位模板，  
并明确标注示例数据、待替换信息与真实证明材料清单，帮助你把“大创材料”从一堆碎片推进到可继续精修的完整草稿。

## 安装

### 方式 1：使用 Codex skill-installer 直接安装

```powershell
python "${HOME}\.codex\skills\.system\skill-installer\scripts\install-skill-from-github.py" --repo star-plucking/dachuang --path . --name dachuang
```

### 方式 2：手动克隆到本地 skills 目录

```powershell
git clone https://github.com/star-plucking/dachuang.git "${HOME}\.codex\skills\dachuang"
```

安装后重启 Codex，即可让它发现这个 skill。

如果你使用 Claude Code，也可以把仓库放到项目中复用 `SKILL.md`、`references/` 和 `assets/` 中的工作流说明与模板材料。

## 快速使用方式

推荐优先在 Codex 中使用本 skill。

最简单的使用方法：

1. 把本项目 GitHub 链接复制给 Codex，让它帮你安装这个 skill。
2. 安装成功后，在 Codex 中打开你的项目文件夹或材料目录。
3. 直接对 Codex 说：`使用 dachuang skill，帮我生成一套大创材料`
4. 然后按提示继续提供项目方向、学校模板、项目级别、经费额度、团队成员和已有进展。

建议你在第一次使用时优先提供：

- 项目方向或已有题目
- 学校申报书 / 开题报告模板
- 项目级别、周期和经费额度
- 团队成员分工与指导老师信息
- 已完成的代码、原型、调研、实验或竞赛材料

这样 Codex 才能按“先定题，再匹配模板，最后生成分阶段材料包”的正确流程工作。

## 推荐搭配 Skills

以下搭配适合大创长流程材料整理：

- `dachuang`
  大创主流程，负责选题策划、申报开题、中期检查、结题验收、答辩展示和成果包装
- `planning-with-files`
  用于管理多阶段材料，避免在申报、中期、结题和答辩之间丢失状态
- `verification-before-completion`
  在声称“材料已经生成好”之前，检查占位数据、待替换字段、附件清单和真实性边界
- `docx`
  当学校要求 Word 表格或 `.docx` 成稿时，用于读取模板、套用样式和检查文档结构
- `playwright`
  当需要抓系统原型截图、演示流程截图或项目运行界面时使用

如果项目还涉及政策背景、行业动态或最新研究现状，也可以让 Codex 联网检索公开资料，但引用和背景需要来自可靠来源，不要编造文献。

## 设计原则

1. 选题要像大创项目，不只是论文题目。
2. 四字主标题要有记忆点，副标题要说明真实技术和应用场景。
3. 项目事实优先，不凭空声称真实成果。
4. 申报、开题、中期、结题、答辩和成果证明要形成闭环。
5. 示例数据必须清楚标注，真实提交前必须替换。
6. 降 AI 味不是编故事，而是减少空话、套话和过度包装。
7. AI 生成图片只能作为概念示意图、封面图或占位视觉，不能冒充真实实验照片、系统截图或成果证明。
8. 最终对用户展现过程与结果默认使用中文。

## 推荐工作流

建议按下面顺序使用本 skill：

1. 明确项目方向、学科背景、项目级别、周期和团队情况。
2. 生成 3-8 个大创风格候选题目。
3. 对候选题目做可行性、数据来源、预算、展示效果和答辩风险评分。
4. 选定“四字主标题——具体项目说明”的最终题目。
5. 按学校模板生成申报书或开题报告。
6. 用评审视角审查材料，压掉空泛表述和不可信承诺。
7. 规划月度任务、会议记录、实验日志和风险登记表。
8. 生成中期报告，体现阶段成果、问题与调整。
9. 生成结题报告，补齐系统实现、测试分析、经费总结和成果说明。
10. 生成答辩 PPT 提纲、讲稿、演示流程和常见问题回答。
11. 对全文做降 AI 味润色和材料一致性检查。
12. 输出真实信息替换清单、证明材料清单和诚信边界提醒。

## 常用脚本

### 1. 检查材料 AI 味和模板腔

```bash
python3 tools/audit_ai_tone.py draft.md
```

脚本会标出常见空泛短语、口号式表达和缺少项目细节的句子。它只做启发式检查，最终仍以真实项目事实和学校模板要求为准。

### 2. 检查多份材料一致性

```bash
python3 tools/check_consistency.py proposal.md midterm.md final.md defense.md
```

脚本会尝试抽取项目题目、指导老师、项目级别、经费总额、项目周期等字段，并提示可能冲突的位置；同时提醒成果、获奖、软著、专利和数据描述是否可能缺少 `示例` / `占位` / `拟申请` 等标注。

### 3. 可选生成项目配图

本仓库提供一个可选绘图脚本，支持 OpenAI GPT Image 系列或用户自定义的 Nano Banana 2 兼容接口。API Key 需要用户手动配置，仓库不会内置任何密钥。

```bash
cp .env.example .env
```

编辑 `.env` 后再运行：

```bash
python3 tools/draw_project_image.py --provider openai --prompt prompt.txt --out output/project-cover.png
```

或使用 Nano Banana 2 / 第三方兼容接口：

```bash
python3 tools/draw_project_image.py --provider nano-banana2 --prompt prompt.txt --out output/project-cover.png
```

生成图片适合用于 PPT 封面、海报背景、项目概念图和占位示意图。不要把 AI 生成图当作真实实验照片、真实系统截图、证书、票据或已完成实物证明。

## 数据与成果边界

Skill 可以生成详细、真实感较强的参考数据表、成果描述、软著 / 竞赛 / 获奖占位写法，但这些内容必须标注为 `参考示例` 或 `占位示例`。

真实提交前需要替换：

- 实验日志、测试记录、截图和视频
- 经费票据和财务系统数据
- 真实论文、软著、专利、竞赛和获奖证明
- 真实系统截图、实物照片、实验现场照片和演示视频
- 指导教师意见、签字、盖章和学校要求字段

不要把占位示例当作真实结果提交。
